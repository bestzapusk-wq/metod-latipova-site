# Метод Латипова — мини-апп MVP

Минимальный Telegram Mini App: каталог курсов с видео-уроками, кнопка
"Сообщество" (ссылка на группу) и форма заявки "База материалов",
которая шлёт заявки тебе в Telegram. Без базы данных, без авторизации,
без оплаты — на первую версию этого не нужно.

## Структура проекта

```
data/courses.json        ← курсы и уроки (редактируешь руками, без БД)
lib/telegram.js           ← отправка сообщений в Telegram Bot API
pages/index.js             ← главная (профиль + 3 кнопки + сетка курсов)
pages/courses/[id].js     ← страница курса со списком уроков и видео
pages/apply.js             ← форма заявки "База материалов"
pages/api/apply.js         ← серверная функция, шлёт заявку в Telegram
styles/globals.css         ← вся стилизация (тёмная тема, лаймовый акцент)
```

## 1. Установка

Нужен Node.js 18+. Открой проект в Cursor, в терминале:

```bash
npm install
```

## 2. Создай Telegram-бота

1. Напиши @BotFather → `/newbot` → следуй инструкциям, получишь токен вида
   `123456789:AAExampleToken...`
2. Узнай chat_id, куда будут падать заявки:
   - Напиши @userinfobot — он покажет твой личный chat_id, или
   - Создай отдельный админ-чат, добавь туда бота, отправь любое сообщение
     и открой в браузере `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
     — там будет `chat.id`.

## 3. Заполни переменные окружения

Скопируй `.env.example` в `.env.local` и заполни:

```bash
cp .env.example .env.local
```

```
TELEGRAM_BOT_TOKEN=токен из BotFather
TELEGRAM_ADMIN_CHAT_ID=chat_id из шага 2
NEXT_PUBLIC_COMMUNITY_URL=ссылка на твою группу с темами
```

## 4. Наполни курсы и видео

Открой `data/courses.json`. Для каждого урока:

1. Залей видео на YouTube со статусом **"Без списка" (Unlisted)** —
   не появится в поиске, но откроется по прямой ссылке/embed.
2. Скопируй ID видео из ссылки
   (`https://www.youtube.com/watch?v=ABC123` → ID это `ABC123`).
3. Вставь его в поле `youtubeId` вместо `PUT_YOUTUBE_ID_HERE`.

Можно добавлять/удалять курсы и уроки — это просто JSON, без БД и миграций.

## 5. Запуск локально

```bash
npm run dev
```

Откроется на `http://localhost:3000` — выглядит и работает как обычный сайт.
Чтобы проверить именно внутри Telegram, нужен публичный HTTPS-адрес
(см. шаг 6) — без него `Telegram.WebApp` API частично недоступен,
но вёрстку и логику можно проверять прямо в браузере.

## 6. Деплой на Vercel

1. Залей проект на GitHub.
2. Зайди на vercel.com → New Project → выбери репозиторий.
3. В настройках проекта (Settings → Environment Variables) добавь те же
   переменные, что в `.env.local`.
4. Deploy. Получишь адрес вида `https://cashew-miniapp.vercel.app`.

## 7. Подключи мини-апп к боту

В @BotFather:

```
/mybots → выбери бота → Bot Settings → Menu Button → Edit Menu Button URL
```

Вставь свой адрес с Vercel. Теперь при открытии бота в Telegram будет
кнопка, открывающая мини-апп.

## Что дальше (когда понадобится)

- **Проверка подписки** — если решишь ограничивать доступ к курсам,
  понадобится база (например, Supabase) и проверка
  `Telegram.WebApp.initDataUnsafe.user.id` на сервере.
- **Избранное** — тоже потребует базы (хранить, что юзер лайкнул).
- **Чаты по городам, бонусы** — можно добавить аналогично "Сообществу"
  (просто кнопки-ссылки) или отдельными страницами, когда будет ясен скоуп.
